boxBar
v0.1.4
description
boxBar is a Blackbox for Windows plugin that provides a system bar. It is designed to be easily extensible, and is becoming more configurable. Its current major feature is that allows a vertical bar.

boxBar has been tested with boxCore on Windows Vista x64 by the author, and on Windows XP by users. It should also work under other branches (except xoblite at this point, due to task support), and has had some testing under vanilla bbLean.

Enjoy using boxBar

initial setup
To load boxBar, just add the relative path of the plugin (from your Blackbox directory) to the Blackbox configuration file
plugins.rc
(located in the Blackbox directory). For example, if you installed Blackbox into C:\Blackbox, and boxBar.dll is located in C:\Blackbox\Plugins\boxBar, you should add the line
Plugins\boxBar\boxBar.dll
to plugins.rc. Simple, eh? :)

boxBar.rc settings
boxBar reads an rc file with the same name as the plugin. So if you leave it as boxBar.dll it will read boxBar.rc. If you rename the plugin to otherBar.dll then it will read otherBar.rc. The setting names within the rc file do not change with the plugin name, only the name of the file read

Currently, boxBar supports the following settings which are intended to be changed by users. They are included in the default rc file.

boxBar.items: see below
    Items which appear in the bar, described in detail in the items section

boxBar.vertical: true|false
    Is the bar vertical or horizontal

boxBar.percentage: 0-100
    The percentage of the monitor to use

boxBar.toggleWithPlugins: true|false
    Toggle visibility with plugins

boxBar.useSlit: true|false
    Plugin in slit if available

boxBar.tasks.iconsize: 1-256
   Size of icons for tasks (16 or 32 looks good)

boxBar.tray.iconsize: 1-256
   Size of tray icons (16 looks best)

boxBar.tray.vertical: true|false
   Sets the tray to vertical or horizontal.

boxBar.tray.maxCols: 0-256
   Maximum number of icons per row. (0 disables)

boxBar.tray.maxRows: 0-256
   Maximum number of icons per column. (0 disables)

boxBar.tray.newIconsfirst: true|false
   Changes the order in which tray icons are displayed

boxBar.clock.format: %H:%M
   What is displayed on a clock label

boxBar.clock.tipFormat: %A %d %B %Y
   What is displayed on the clock tooltip

boxBar also supports the following settings, but use them with caution.

boxBar.drawBackground: true|false
   Switches main bar style to none. Requires 2k/XP or better

boxBar.useAlphaBlend: true|false
   Enables alphablending draw when available (Win 98+)

boxBar.tasks.stretchArea: true|false
   When false, task section does not expand. Experimental

boxBar.spacingBorder: 0-100
   Bar border spacing

boxBar.spacingItems: 0-100
    Bar inter-item spacing

boxBar.tasks.spacingBorder: 0-100
    Taskarea border spacing

boxBar.tasks.spacingItems: 0-100
    Taskarea inter-item spacing

boxBar.tasks.task.spacingBorder: 0-100
    Border spacing inside a task

boxBar.tasks.task.spacingItems: 0-100
    Gap between task icon and text

boxBar.tray.spacingBorder: 0-100
    Tray border spacing

boxBar.tray.spacingItems: 0-100
    Tray area inter-item spacing

boxBar items
You can use as many or as few of the following item to make up your boxBar.items: setting. Duplicates are allowed, but do not use spaces. The default is:

boxBar.items: tasks,tray,clock
   

•clock
A clock label on your bar. It is configured by settings starting boxBar.clock

•tasks
A section which contains the current tasks. You can adjust settings for this by changing settings starting with boxBar.tasks. This section will expand to fill as much space as possible. Leftclick activates a task.

•tray
A system tray area. Will display all your tray icons. Can be set to have a speicifc number of rows or columns as well as opther settings through the settings starting with boxBar.tray.

•wslabel
A label to indicate your current workspace. Leftclick goes to next, rightclick to previous.

additional notes
This plugin will not create, delete or edit any files at all except for the boxBar.rc file.
